# Quickstart

Get vecna running in your project.

## Installation

```bash
go get github.com/zoobzio/vecna
```

Requires Go 1.24 or higher.

## Basic Usage

```go
package main

import (
    "fmt"
    "log"

    "github.com/zoobzio/vecna"
)

type Metadata struct {
    Category string  `json:"category"`
    Score    float64 `json:"score"`
    Active   bool    `json:"active"`
}

func main() {
    // Create a schema-validated builder
    builder, err := vecna.New[Metadata]()
    if err != nil {
        log.Fatal(err)
    }

    // Build a filter with nested logic
    filter := builder.And(
        builder.Where("category").Eq("tech"),
        builder.Or(
            builder.Where("score").Gte(0.8),
            builder.Where("active").Eq(true),
        ),
    )

    // Check for construction errors
    if err := filter.Err(); err != nil {
        log.Fatal(err)
    }

    fmt.Printf("Built filter: %s\n", filter.Op())
}
```

## Deferred Errors

vecna doesn't panic on invalid fields. Errors are stored in the filter and surface when you call `Err()`:

```go
filter := builder.Where("categroy").Eq("tech")  // typo
// No panic here - filter is constructed

if err := filter.Err(); err != nil {
    // Error: vecna: field not found: categroy
}
```

This allows building complex filter trees without checking errors at each step. See [Concepts](3.concepts.md) for more on error handling.

## Filter Specs

Filters can be built from JSON for dynamic queries:

```go
var spec vecna.FilterSpec
json.Unmarshal([]byte(`{"op":"eq","field":"category","value":"tech"}`), &spec)

filter := builder.FromSpec(&spec)
```

See the [Specs Guide](../2.guides/1.specs.md) for complete coverage.

## Operators

All comparison and logical operators:

```go
builder.Where("field").Eq(value)      // Equal
builder.Where("field").Ne(value)      // Not equal
builder.Where("field").Gt(value)      // Greater than (numeric)
builder.Where("field").Gte(value)     // Greater than or equal (numeric)
builder.Where("field").Lt(value)      // Less than (numeric)
builder.Where("field").Lte(value)     // Less than or equal (numeric)
builder.Where("field").In(a, b, c)    // Set membership

builder.And(filter1, filter2)         // Logical AND
builder.Or(filter1, filter2)          // Logical OR
```

See [Operators Reference](../4.reference/2.operators.md) for details.
